How I Built an Interactive Resume Web App in 1 Day with Claude Code, Gemini 3.1 Pro, and modern AI tooling

Dominik Benger February 2026 15 min read

Summary

This case study documents how I built dbenger.com — a fully interactive resume web app — in approximately 1 day using Claude Code as my primary development partner. The site replaces a static 4-page PDF with animated SVG visualizations, a filterable career timeline, an AI chatbot, and a service offerings page.

The project was built with Next.js 15, Tailwind CSS, Framer Motion, and custom SVG visualizations (no charting library). The AI chatbot runs on Gemini 3 Flash Preview via Google AI Studio with SSE streaming. The entire site ships with just 6 production dependencies and 158 kB of first-load JavaScript.

Visual mockups and UI exploration were prototyped using Gemini 3.1 Pro for rapid design iteration before implementation, enabling high-fidelity design decisions without traditional design tools.

Project Overview

The Problem

Static PDF resumes are limited. They're flat, one-dimensional documents that force 10+ years of career experience into bullet points on a page. Recruiters scan them in 30 seconds. There's no way to filter by relevance, explore career progression visually, or engage with the content interactively.

The Vision

Replace my PDF resume with a fully interactive web application — a living document where visitors could:

  • Explore my career trajectory through animated SVG visualizations
  • Filter my experience by industry, role type, or technical capability
  • Interact with 77 skills across 7 categories in a filterable tag grid
  • Connect directly through an integrated contact form
  • Download a traditional PDF when they needed one

The Constraint

I built the entire project in approximately 1 day using Claude Code as my primary development partner. No design team, no Figma mockups — instead, I used Gemini 3.1 Pro to rapidly prototype UI mockups and explore visual directions before implementation. Just a 4-page resume PDF, a product requirements document, and AI tools.

What I Didn't Use (and Why)

✕No charting library — Custom SVG gave full control without bundle bloat
✕No CMS — Career data lives in TypeScript files. A CMS adds complexity without value.
✕No test framework — The site is entirely presentational. Visual QA was sufficient.
✕No database — Everything is statically generated at build time.

Key Insight: The "no charting library" decision saved significant bundle size and gave pixel-perfect control over the Career Path visualization — including year-proportional node positioning, custom bezier curves, and per-node label placement.

Development Workflow

Before writing a single line of code, I created three foundational documents:

1

~900 lines covering goals, user stories, functional requirements, UX flows, success metrics. Drafted using ChatPRD and Codex.

2

~380 lines defining the tech stack, data models, design system tokens, component specs, and performance budgets.

~470 lines breaking the project into 32+ tasks across 8 phases with explicit dependencies.

Key Strategy: Front-loading the documentation paid massive dividends. Using ChatPRD and Codex for the PRD, Gemini 3.1 Pro for UI mockups, and Cursor in Plan Mode meant the requirements and visual direction were well-established before any code was written. Claude Code could then reference the spec when making implementation decisions.

The project's CLAUDE.md file grew to ~200 lines and served as persistent memory across sessions — including startup rituals, tech stack reference, architecture decisions, pitfalls, and session continuity notes.

Key Features Built

Feature 1

Career Path Visualization

An animated SVG chart showing career progression from 2014 to 2025. Each node is positioned proportionally by year using cubic bezier curves. The path draws itself using Framer Motion's pathLength animation.

const yearToX = (year: number) =>
  padding.left + ((year - minYear) / yearRange) * chartW
Feature 2

Skills & Tech Stack Grid

A filterable grid of 77 skills across 7 color-coded categories. Clicking a category filter isolates that category; clicking again resets. Colors defined as inline styles via a colorMap object.

Feature 3

Filterable Experience Section

Experience cards filter dynamically using pill toggle buttons with inclusive OR logic. Filter state lives at the page level and flows down to the experience section.

Feature 4

Scroll-Triggered Animations

A custom useScrollAnimation hook using IntersectionObserver. Elements animate once and stay visible. Respects prefers-reduced-motion — bypasses observer entirely.

Feature 5

Collaboration Page

10 service offerings with accordion-style deliverables, 3 packages (Audit/Build/Operate) in a comparison card layout, and a working style section with principle cards.

Feature 6

AI Chatbot ("Ask Dominik's AI")

Powered by Gemini 3 Flash Preview via Google AI Studio with SSE streaming. Uses a curated knowledge base (~5K tokens), in-memory rate limiting (100/day/IP, 20/session), and a manual ReadableStream + TextEncoder for streaming.

Lesson: System prompt instructions like "do not use markdown" are unreliable with LLMs. Always add a client-side stripMarkdown() function as a safety net.

7

Challenges & Solutions

SVG Label Overlap in Year-Proportional Chart

Early-career nodes (2014–2018) bunched in left ~36% of chart

Solution: Per-node labelPlacements array with diagonal offsets. Bunched nodes alternate between above-left and below-right placement.

Mobile Chart Horizontal Scroll

SVG had min-w-[500px] causing horizontal scroll on mobile

Solution: Removed min-width; let SVG scale naturally via viewBox + preserveAspectRatio.

Nav/Filter Bar Z-Index Overlap

Sticky filter bar overlapped the floating nav when scrolling

Solution: Changed filter bar to sticky top-[68px] to sit below the FloatingNav.

Cross-Page Anchor Navigation

#section hrefs didn't work from /collaboration back to home

Solution: Prefix all anchors with / — using /#experience instead of #experience.

Merging Two Visualizations into One

Skills (bar chart) and Tech Stack (timeline) tabs felt incomplete individually

Solution: Merged into a single "Skills & Tech Stack" interactive tag grid with category filters, showcasing all 77 skills at once.

Lessons Learned

On AI-Assisted Development

"Front-load your docs."

The more context you provide upfront (PRD, spec, plan), the less course-correction you need during implementation. The initial autonomous build (10,000+ lines in one commit) was possible because the spec was thorough enough for Claude Code to make correct decisions without asking.

"CLAUDE.md is your most important file."

It's the persistent memory that makes sessions composable. Every pitfall documented, every pattern recorded — they compound across sessions.

"Review everything."

AI-generated code is not reviewed code. The AI got the structure right ~90% of the time; the remaining 10% was the difference between "works" and "polished."

On Architecture

"Fewer dependencies, fewer problems."

Six production dependencies. One API route. Zero database. Every decision to not add something made the project simpler.

"Custom SVG beats chart libraries for bespoke visualizations."

The Career Path chart needed year-proportional positioning, custom bezier curves, per-node label placement, and Framer Motion integration. No charting library offers all of that.

On Design

"Warm palettes create approachability."

The deliberate choice of warm off-white over pure white, and warm near-black over pure black, creates a softer, more inviting feel.

"Use AI for design exploration."

Gemini 3.1 Pro mockups enabled rapid visual iteration — testing color variations, layout compositions, and component designs without writing code. This compressed the design phase from days to hours.

"Navigation simplicity wins."

The FloatingNav went through 3 iterations before arriving at the simplest version: a fixed horizontal pill bar with no scroll triggers, no hamburger menu. The final version is 47 lines.

Final Thought

This project started as a replacement for a 4-page PDF. It became a case study in how AI-assisted development changes the economics of personal projects. What would have been a multi-week effort became a weekend sprint. The tools didn't write the vision — they executed it. The taste, the design decisions, the "should this be a chart or a grid?" choices — those remain human. But the implementation velocity? That's transformed.
